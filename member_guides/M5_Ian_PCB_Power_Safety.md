## Step 4 — PCB layout updates

1. Open PCB editor (`File → Open` → `.kicad_pcb`).
2. **Update from Schematic**: `Tools → Update PCB from Schematic` → tick "Re-link footprints by reference" → Run.
3. New footprints (J6, J7, dividers) appear unrouted.

### 4.1 Layout rules
| Trace | Width |
|---|---|
| Signal | 0.25 mm |
| 3.3 V | 0.40 mm |
| 5 V | 1.0 mm |
| 12 V / motor | 1.5 mm |
| GND | Pour (both sides) |

### 4.2 Antenna keepout
- ESP32-WROOM-32E module antenna: 7 × 30 mm at the top edge.
- File → Board Setup → Design Rules → Custom Rules. Add:
  ```
  (rule "Antenna Keepout"
    (constraint clearance (min 0.5mm))
    (condition "A.intersectsArea('antenna_keepout')"))
  ```
- Or simpler: draw a keepout zone on User.Drawings layer, set "no copper" property.

### 4.3 Place J6 and J7
- J6 (TFT) on the **front edge** of the board, where the TFT cable can run cleanly to the panel-mount bezel.
- J7 (CAM) on the **side**, away from the antenna.

### 4.4 Re-route
- Use 45° corners, not 90°.
- Differential-route MOSI/MISO/SCK if possible.
- Keep digital signals away from motor 12 V trace.

### 4.5 Run DRC (Design Rules Check)
1. Inspect → Design Rules Checker.
2. Fix all violations.
3. Ensure 0 unconnected pads, 0 clearance violations.

## Step 6 — Soldering sequence (when boards arrive)

Solder lowest-profile components first so the board sits flat. Order:

1. **0805 resistors** — flux pad, place R, drag-solder one side, then the other.
2. **0805 capacitors** — same technique.
3. **SOT-23 / SOT-223** — AMS1117, AO3400A, 2N7000.
4. **DIP-16** — 74HC595 (use a socket — cheaper to swap a fried chip).
5. **TVS, polyfuse, Schottky diode**.
6. **Through-hole headers** — pin headers for ESP32 module, J6, J7, J9.
7. **Polarised connectors** — DC jack, JST.
8. **Tall components** — relay last.
9. **ESP32 DevKit module** — push into headers, do **not** solder permanently — you may need to swap it.

After every group: visual inspection under good light + multimeter continuity beep on power rails (should NOT beep between V+ and GND on any rail).

## Step 7 — First power-up procedure

> **Never** plug 12 V into a freshly soldered board cold. Follow this sequence.

1. Continuity check: 12 V to GND, 5 V to GND, 3.3 V to GND — all must be **open** (no beep).
2. With nothing plugged in, set bench supply to 12 V, **current-limited to 100 mA**.
3. Apply power. Current should be < 30 mA. If at limit → power off, find short.
4. Measure with multimeter on the 5 V rail node — should read **5.0 ± 0.1 V**.
5. Measure on the 3.3 V rail — should read **3.30 ± 0.05 V**.
6. Power off. Plug ESP32 DevKit module into headers.
7. Power on (still 100 mA limit). Current ≈ 80 mA — DevKit boots, USB enumerates.
8. Open serial monitor. You should see the boot banner.
9. Raise current limit to 1 A. Test motor command via serial.
10. Raise to 3 A only when running the motor under load.

### 12 V undervoltage tuning
The divider 33k/10k → ADC reads 12 V as 2.79 V (3460 counts). Threshold in `fault_register.cpp` is 11.0 V. To verify:
1. Drop bench supply slowly from 12 V to 10.5 V.
2. At ≈ 11 V the firmware should log `[fault] raised: uv-12v`.
3. Above 11.5 V (with hysteresis margin) the fault clears.

If the threshold is wrong, edit `V12_DIVIDER_RATIO` in `fault_register.cpp` to match your **actual** measured resistor values (use multimeter to measure each before solder).

## Step 8 — Safety firmware deep-dive (already coded — your responsibility to maintain)

### `safety/watchdog.cpp`
- Hardware TWDT: 5 s panic.
- Software watchdog: 5 task kicks (FSM, motor, sensors, vision, main).
- Each task calls `safety_watchdog_kick_*()` periodically.
- `task_safety` polls timestamps at 20 Hz; if any > 1.5 s old → forces SAFE state.
- **Tune** `WATCHDOG_MAX_INTERVAL_MS` only if a task legitimately needs more headroom.

### `safety/fault_register.cpp`
- Reads voltage rails via ADC, evaluates 16 fault flags (bitmask in `g_status.fault_flags`).
- Edge-detects rising/falling fault state, posts `EVT_FAULT_RAISED` / `EVT_FAULT_CLEARED` to FSM.
- `fault_register_first_name(flags)` returns human-readable name for the first set bit.

### `safety/interlocks.cpp`
- E-stop ISR on PIN_ESTOP (CHANGE trigger, INPUT_PULLUP, **HIGH = pressed** because mushroom is NC and opens on press).
- Relay control: energise (HIGH on PIN_RELAY) only if `!estop && fault_flags == 0`.
- Barrier servo control via ESP32Servo lib on PIN_SERVO_L / PIN_SERVO_R.
- Open-loop servo timing — assumes 1 s reach. Real SG90 ≈ 0.6 s for 90°.

### Things you will tune on real hardware
- ADC current calibration (`ADC_TO_MA_NUMERATOR / DENOMINATOR`) in motor_driver.cpp — measured against a known 1 A load.
- Relay flyback diode polarity (orientation matters — cathode goes to V+ side of coil).
- E-stop hard-wire test: cut a wire mid-run → fault should trigger immediately.

## Step 10 — Failure recovery cookbook

| Symptom | Cause | Fix |
|---|---|---|
| 12 V short on first power-up | Bridged trace under SOT-223 reg | Visual under loupe; rework with hot air or fresh iron tip |
| 5 V rail reads 4.6 V | LM2596 trim pot wrong | Adjust trim pot CW until 5.0 V at module output |
| 3.3 V rail oscillates | Missing 10 µF on AMS1117 output | Add 10 µF tantalum or ceramic at output pin |
| ESP32 doesn't enumerate over USB | DevKit module not seated | Re-seat. Ensure pins line up — 19 + 19 row |
| Relay clicks on then immediately off | Coil voltage sag | Confirm 5 V rail can supply 80 mA inrush — bump bulk cap to 1000 µF |
| E-stop trigger doesn't kill motor | Relay contact in wrong line | Verify NO contact in series with **motor V+**, not just signal |
| Watchdog fault every few minutes | Vision task slow | Bump `WATCHDOG_MAX_INTERVAL_MS * 2` for vision (already done — verify) |
| Buzzer silent | LEDC channel collision | Ensure ch0/1 are motor PWM, ch2 backlight, ch5 buzzer — all unique |
